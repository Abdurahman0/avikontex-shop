# AvikonCRM — Bugungi ish hisoboti (2026-07-06)

Enterprise CRM+ERP+Marketplace platformasiga **auth, kompaniya, RBAC, marketplace
(mahsulot/savat/liked)** modullari qo'shildi. Barchasi **lokal (8010, sqlite)** va
**docker (8000, postgres)** da testlandi.

Umumiy tamoyillar: Clean architecture, SOLID, service/repository qatlami,
transaction, validation, **BaseModel** (uuid + audit + soft delete), DB-driven RBAC.

---

## 0. BaseModel (poydevor)
`common/base.py` — barcha yangi modellar shundan meros oladi:
- `id` (UUID PK), `created_at`, `updated_at`
- `created_by`, `updated_by` (audit)
- `is_deleted`, `deleted_at` + `SoftDeleteManager` (soft delete) va `soft_delete()/restore()`

## FAZA 1 — Identity & Auth

### Modellar
- **User** kengaytirildi (`users/models.py`): `phone` (unique), `status`
  (PENDING_SMS / ACTIVE / BLOCKED), `account_type` (jismoniy / yuridik). Mavjud
  staff buzilmadi (username saqlanadi).
- **Person** — passport (FIO, seriya, raqam, PINFL, tug'ilgan sana, jins), `user` OneToOne.
- **OtpCode** — telefon, kod, muddat, is_used, attempts.

### OTP (dev rejim)
`users/otp.py` + `config/settings.py`:
- Hozir **`OTP_DEV_MODE=True`** → kod doim **`000000`**, haqiqiy SMS yuborilmaydi (log).
- Real SMS'ga o'tish: `.env` da `OTP_DEV_MODE=false` → tasodifiy kod + `_send_sms` provayder (Eskiz/Play Mobile) ga ulanadi.

### Oqim
```
POST /api/auth/register/       jismoniy: passport + OTP → User(PENDING_SMS)+Person+Client(jismoniy)
POST /api/auth/register/company/  yuridik: passport + kompaniya → Client(yuridik)+Company+User(PENDING_SMS)
POST /api/auth/verify-otp/     { phone, code:"000000" } → ACTIVE + JWT
POST /api/auth/resend-otp/     { phone }
POST /api/auth/login/          { login: email/telefon, password } (PENDING login qila olmaydi → 403)
GET  /api/auth/me/             account_type, status, client (INN/company bilan)
```

## FAZA 2 — Company + DB-driven RBAC

### Modellar (`companies` app)
- **Company** — company_type (YaTT/MChJ), legal_name, inn (unique), settlement_account,
  bank, mfo, legal_address, real_address, director (FK Person), owner (FK User).
- **CompanyMember** — bridge (company, user, role, status, invited_by, joined_at). Direct FK yo'q.
- **Role / Permission / RolePermission** — DB orqali boshqariladi (hardcode emas).
- **Invitation** — phone/email + token + role + muddat.

### Seed
`python manage.py seed_rbac` → **14 permission**, **9 system rol**
(Owner, Director, Admin, Manager, Operator, Warehouse, Accountant, Sales, Courier).

### Endpointlar
```
POST /api/companies/                     kompaniya yaratish (yaratuvchi = Owner)
GET  /api/companies/                     a'zo bo'lgan kompaniyalar
GET  /api/companies/<id>/                detali (faqat a'zo)
GET  /api/companies/<id>/members/        a'zolar (rol bilan)
POST /api/companies/<id>/invite/         taklif (members.manage kerak)
POST /api/companies/invitations/accept/  taklifni qabul (token)
GET/POST /api/roles/                     system + kompaniya rollari
GET  /api/permissions/                   barcha permissionlar
```
RBAC enforcement testlandi: `members.manage` yo'q a'zo taklif qila olmaydi → 403.

## FAZA 3 — Documents, Branch, Settings, References

### `references` app (yangi)
- **Bank**, **Region**, **District**, **Address** + seed:
  `python manage.py seed_references` → 15 bank, 14 viloyat, 6 hujjat turi.

### `companies` app (qo'shildi)
- **CompanyBranch** — filiallar (company, name, address FK, phone, is_main).
- **CompanySettings** — til/valyuta/vaqt zonasi (kompaniya yaratilganda avtomatik).
- **DocumentType** + **CompanyDocument** — hujjat yuklash (multipart file upload).

### Endpointlar
```
GET  /api/references/banks/               banklar (autocomplete)
GET  /api/references/regions/             viloyatlar + tumanlar
GET  /api/document-types/                 hujjat turlari
GET/POST /api/companies/<id>/branches/    filiallar
GET/PUT  /api/companies/<id>/settings/    sozlamalar
GET/POST /api/companies/<id>/documents/   hujjatlar (file upload)
```

### CompanyAddress — ko'p tipli manzillar
Bir kompaniyada **bir nechta manzil**: yuridik / real / dostavka / to'lov / ombor.
```
GET/POST /api/companies/<id>/addresses/            (?type=delivery filtr)
GET/PUT/DELETE /api/companies/<id>/addresses/<addr_id>/
```
- Strukturali (region, district, street, house, apartment, postal_code, landmark)
- Dostavka uchun: recipient_name, recipient_phone
- `is_default` — har bir tur uchun **bitta** asosiy (yangisi eskisini almashtiradi)
- O'chirish = soft delete

## Marketplace: mahsulotlar + savat + liked

### Public mahsulot API (tokensiz)
`unf/views.py` — quyidagilar **AllowAny** (login shart emas):
```
GET /api/unf/products/            (barcha filtr: search, group, segment, in_stock, min/max_price, ...)
GET /api/unf/product-groups/
GET /api/unf/product-segments/
```
> `customers`, `tables` (xom 1C) himoyalangan qoldi (401).

### Mahsulot rasmi (host-agnostic)
- `imageUrl` **request host'dan** quriladi (`unf/products.py`, `unf/views.py`) —
  ngrok URL o'zgarsa o'zi to'g'rilanadi.
- `SECURE_PROXY_SSL_HEADER` + `USE_X_FORWARDED_HOST` — ngrok orqasida **https** to'g'ri.
- Barcha mahsulot hozircha `media/products/00000000001.jpg` ni ishlatadi.

### Liked products + savat (`clients` domenida)
Muhim qaror: **cart/favorites `clients` app'da, egalik = Client** (shop asosiy emas,
clients asosiy). Jismoniy register ham endi **Client yaratadi** (nomuvofiqlik tuzatildi).

Modellar (`clients/models.py`): **FavoriteItem**, **CartItem** — egalik `client` (FK)
**yoki** `device_id` (guest). Mahsulot 1C `code` orqali (FK emas), narx **jonli**.

```
GET    /api/clients/favorites/               (jonli narx/rasm)
POST   /api/clients/favorites/  {product_code}
DELETE /api/clients/favorites/<code>/
GET    /api/clients/cart/                     (items + jami)
POST   /api/clients/cart/items/  {product_code, quantity}
PATCH  /api/clients/cart/items/<code>/  {quantity}
DELETE /api/clients/cart/items/<code>/
```

**Guest → Client merge:** frontend `X-Device-Id` header yuboradi (localStorage UUID).
Register/verify/login'da o'sha device_id yuborilsa, guest savat/likes avtomatik
mijoz Client'iga ko'chadi (favorites dedup, cart qty birlashadi). `shop` app o'chirildi.

## CORS (frontend uchun)
`config/settings.py`:
- `CORS_ALLOW_CREDENTIALS=True`, `CORS_ALLOWED_ORIGIN_REGEXES` (localhost, 127.0.0.1, *.ngrok)
- `CORS_ALLOW_HEADERS`: default + **`ngrok-skip-browser-warning`** + **`x-device-id`**
- `CSRF_TRUSTED_ORIGINS`: ngrok domenlari
- Aniq origin echo qilinadi (`*` emas) → credentials bilan ishlaydi.

## Operatsion eslatmalar
1. **1C bazasi konteyneri `avikon_pg`** — docker-compose'dan tashqarida. To'xtasa
   `docker start avikon_pg` + `docker network connect avikoncrm_default avikon_pg`.
   (Bugun to'xtab qolgani uchun productlar 500 bergandi — tuzatildi.)
2. **OTP** hozir `000000`. Real SMS: `OTP_DEV_MODE=false`.
3. **ngrok bepul** — API uchun `ngrok-skip-browser-warning` header kerak; `<img>` rasmlar
   buzuq (header qo'sha olmaydi). To'liq yechim: **cloudflared** (ogohlantirishsiz).
4. Migratsiyalar qo'llangandan keyin: `seed_rbac` + `seed_references` (docker'da ham).
